Описание.
-------------------------------
Программа для продления срока работы игр от компании Alawar.
Возможности программы:
— Регистрация игры;
— Сброс триала;
— Крек игры. 

Использование.
-------------------------------
1: Выбрать один из пунктов:
  - Register game - регистрация игры. Заносит в реестр верный SMS ключ. Изменений в файле игры не производится. 
  - Clean trial - сброс триала. Возобновляет в реестре триальный период. Изменений в файле игры не производится. 
  - Crack game - крек игры. Исправляет в оригинальном файле игры "неверные" байты. По желанию пользователя, возможна автоматическая замена файла-загрузчика файлом игры. Изменений в реестре не производится. 
2: Нажать кнопку Start.
3: Выбрать файл игры (файл, на который указывает ярлык).
4: Дождаться результатов.

История.
-------------------------------
 - v2.1 Доработан алгоритм поиска ключей реестра для сброса триала и регистрации.
 - v2.0 Программа переписана с нуля. Реализована поддержка новых версий защиты правкой файла AlawarDie.dat.
 - v1.0 Писалась для себя, поддерживалась только старая версия защиты, в свет так и не вышла.

Структура файла AlawarDie.dat.
-------------------------------
DWORD Version;                Версия (DWORD по смещению 0h)
WORD  StolenBytesRVAOffset;   Смещение RVA краденых байт
WORD  StolenBytesOffset;      Смещение краденых байт
WORD  StolenBytesCountOffset; Смещение количества краденых байт
WORD  GameFileOffset;         Смещение имени оригинального файла игры **
WORD  TrialTimeOffset;        Смещение триального времени *
WORD  TrialKeyOffset;         Смещение ключа реестра, хранящего триал **
WORD  GameKeyOffset;          Смещение ключа реестра, хранящего регистрационный ключ **
WORD  Reserved;               Добавлено на всякий случай (лучше заполнить нулями)

 *  - толком не понял то ли это значение, смещение всегда 30h
 ** - смещение лучше указывать с небольшим количеством нулей перед строкой, т.к. эти смещения немного отличаются в одной и той-же версии, нули отсекаются автоматически.

Структуры могут быть записаны в файл последовательно. При присутствии нескольких структур с одинаковыми версиями в расчёт принимается только первая из них!
